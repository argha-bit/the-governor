# API Gateway Configuration Handler - Template Specification v1.0

## 1. SERVICE REGISTRATION REQUEST

### POST /api/v1/services/register

**Request Body:**
```json
{
  "service_name": "user-service",
  "team_name": "platform-team",
  "team_contact": "platform-team@example.com",
  "environment": "production",
  "routing_config_endpoint": "https://github.com/platform-team/user-service/raw/main/gateway-config.json",
  "routing_config_auth": {
    "type": "bearer",
    "token_secret_ref": "github-token"
  },
  "webhook_url": "https://user-service.internal/webhooks/gateway-sync",
  "webhook_secret": "webhook-secret-key",
  "metadata": {
    "owner": "john.doe@example.com",
    "repository": "https://github.com/platform-team/user-service",
    "service_version": "v2.3.1"
  }
}
```

**Response:**
```json
{
  "service_id": "srv_abc123def456",
  "status": "registered",
  "registered_at": "2024-03-05T10:30:00Z",
  "next_sync_at": "2024-03-05T10:35:00Z",
  "validation": {
    "config_endpoint_reachable": true,
    "webhook_verified": true
  }
}
```

## 3. WEBHOOK REQUEST (Sent by Gateway to Service)

### POST {service.webhook_url}

**Headers:**
```
X-Gateway-Signature: sha256=<hmac_signature>
X-Gateway-Event: config.sync.requested
X-Gateway-Delivery: evt_123456
```

**Request Body:**
```json
{
  "event": "config.sync.requested",
  "service_id": "srv_abc123def456",
  "service_name": "user-service",
  "team_name": "platform-team",
  "timestamp": "2024-03-05T10:35:00Z",
  "sync_reason": "scheduled",
  "previous_sync_at": "2024-03-05T10:30:00Z"
}
```

**Expected Response:**
```json
{
  "status": "acknowledged",
  "config_version": "v2.3.1",
  "config_endpoint": "https://github.com/platform-team/user-service/raw/main/gateway-config.json"
}
```

## 4. DATABASE SCHEMA

### Table: services

```sql
CREATE TABLE services (
    service_id VARCHAR(50) PRIMARY KEY,
    service_name VARCHAR(100) NOT NULL,
    team_name VARCHAR(100) NOT NULL,
    team_contact VARCHAR(255),
    environment VARCHAR(50) DEFAULT 'production',
    routing_config_endpoint TEXT NOT NULL,
    routing_config_auth_type VARCHAR(50),
    routing_config_auth_secret_ref VARCHAR(255),
    webhook_url TEXT,
    webhook_secret VARCHAR(255),
    status VARCHAR(50) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_sync_at TIMESTAMP,
    next_sync_at TIMESTAMP,
    sync_interval_seconds INT DEFAULT 300,
    metadata JSONB,
    UNIQUE(service_name, team_name, environment)
);

CREATE INDEX idx_services_team ON services(team_name);
CREATE INDEX idx_services_status ON services(status);
CREATE INDEX idx_services_next_sync ON services(next_sync_at);
```

### Table: routes

```sql
CREATE TABLE routes (
    route_id VARCHAR(50) PRIMARY KEY,
    service_id VARCHAR(50) REFERENCES services(service_id) ON DELETE CASCADE,
    route_config JSONB NOT NULL,
    enabled BOOLEAN DEFAULT true,
    priority INT DEFAULT 100,
    hostnames TEXT[],
    path_prefix VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_applied_at TIMESTAMP,
    apply_status VARCHAR(50) DEFAULT 'pending',
    apply_error TEXT,
    checksum VARCHAR(64)
);

CREATE INDEX idx_routes_service ON routes(service_id);
CREATE INDEX idx_routes_enabled ON routes(enabled);
CREATE INDEX idx_routes_hostnames ON routes USING GIN(hostnames);
CREATE INDEX idx_routes_checksum ON routes(checksum);
```

### Table: sync_history

```sql
CREATE TABLE sync_history (
    sync_id SERIAL PRIMARY KEY,
    service_id VARCHAR(50) REFERENCES services(service_id) ON DELETE CASCADE,
    sync_started_at TIMESTAMP NOT NULL,
    sync_completed_at TIMESTAMP,
    sync_status VARCHAR(50),
    routes_added INT DEFAULT 0,
    routes_updated INT DEFAULT 0,
    routes_deleted INT DEFAULT 0,
    config_version VARCHAR(100),
    error_message TEXT,
    fetch_duration_ms INT,
    apply_duration_ms INT
);

CREATE INDEX idx_sync_history_service ON sync_history(service_id);
CREATE INDEX idx_sync_history_started ON sync_history(sync_started_at DESC);
```

### Table: webhook_deliveries

```sql
CREATE TABLE webhook_deliveries (
    delivery_id VARCHAR(50) PRIMARY KEY,
    service_id VARCHAR(50) REFERENCES services(service_id) ON DELETE CASCADE,
    event_type VARCHAR(100) NOT NULL,
    payload JSONB NOT NULL,
    attempt_count INT DEFAULT 0,
    last_attempt_at TIMESTAMP,
    next_attempt_at TIMESTAMP,
    status VARCHAR(50) DEFAULT 'pending',
    response_status_code INT,
    response_body TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_webhook_service ON webhook_deliveries(service_id);
CREATE INDEX idx_webhook_status ON webhook_deliveries(status);
CREATE INDEX idx_webhook_next_attempt ON webhook_deliveries(next_attempt_at);
```

### Table: audit_log

```sql
CREATE TABLE audit_log (
    log_id SERIAL PRIMARY KEY,
    service_id VARCHAR(50),
    action VARCHAR(100) NOT NULL,
    actor VARCHAR(255),
    details JSONB,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_audit_service ON audit_log(service_id);
CREATE INDEX idx_audit_timestamp ON audit_log(timestamp DESC);
```

## 5. CONFIGURATION FIELD REFERENCE

### Route Priority
- Lower number = higher priority
- Range: 1-1000
- Default: 100

### Path Match Types
- `PathPrefix` - Matches path prefix (e.g., `/api` matches `/api/users`)
- `Exact` - Exact path match
- `RegularExpression` - Regex pattern (advanced)

### Header/Query Match Types
- `Exact` - Exact value match
- `RegularExpression` - Regex pattern
- `Present` - Header/param exists (any value)

### Backend Types
- `kubernetes_service` - Service in same cluster
- `external_service` - External hostname/IP
- `weighted_backends` - Traffic splitting (canary, A/B)
- `aws_lambda` - AWS Lambda function (future)
- `azure_function` - Azure Function (future)

### Protocol Types
- `http` - Plain HTTP
- `https` - HTTPS/TLS
- `grpc` - gRPC
- `http2` - HTTP/2

### Retry On Conditions
- `5xx` - Server errors
- `gateway-error` - Gateway errors
- `reset` - Connection reset
- `connect-failure` - Connection failed
- `refused-stream` - Stream refused

### Rate Limit Dimensions
- `header` - Based on header value
- `remote_address` - Client IP
- `generic_key` - Custom key

## 6. VALIDATION RULES

### Required Fields (Minimal Route)
```json
{
  "route_id": "required-string-unique-per-service",
  "hostnames": ["required-array-min-1"],
  "path_config": {
    "type": "required-PathPrefix|Exact|RegularExpression",
    "value": "required-string"
  },
  "backend": {
    "type": "required-kubernetes_service|external_service|etc"
  }
}
```

### Field Constraints
- `route_id`: 1-100 chars, alphanumeric + dash/underscore
- `hostnames`: Valid DNS hostnames
- `priority`: 1-1000
- `timeout.request_timeout`: Valid duration (e.g., "30s", "1m")
- `port`: 1-65535
- `weight`: 1-100 (for weighted backends)

## 7. TEMPLATE DECISION: Why JSON?

### Reasons for JSON:
✅ **Universal** - Every language has JSON support
✅ **Validation** - JSON Schema for strict validation
✅ **Versioning** - Easy to version and migrate
✅ **Tooling** - IDEs, linters, formatters available
✅ **Git-friendly** - Readable diffs
✅ **API-native** - Direct HTTP payload

### Alternative Considered: YAML
- More human-readable
- Less strict (can cause issues)
- Harder to validate programmatically

### Recommendation: **JSON with JSON Schema validation**

## 8. USAGE FLOW

```
1. Team registers service
   POST /api/v1/services/register
   
2. Gateway fetches config from team's endpoint
   GET {routing_config_endpoint}
   
3. Gateway validates config against JSON Schema
   
4. Gateway converts to Kubernetes Gateway API objects
   
5. Gateway applies HTTPRoute/Gateway to cluster
   
6. Gloo discovers and creates Upstreams
   
7. Traffic flows through configured routes
   
8. On changes, gateway sends webhook to team
   POST {webhook_url}
```

---

## 9. EXAMPLE: Complete Registration Flow

### Step 1: Team registers
```bash
curl -X POST http://gateway-service/api/v1/services/register \
  -H "Content-Type: application/json" \
  -d @register-request.json
```

### Step 2: Team provides gateway-config.json
At `https://github.com/team/service/raw/main/gateway-config.json`

### Step 3: Gateway syncs every 5 minutes
- Fetches config
- Validates
- Converts to HTTPRoute
- Applies to cluster

### Step 4: Team gets webhook on sync
```json
{
  "event": "config.sync.completed",
  "routes_applied": 5,
  "status": "success"
}
```
